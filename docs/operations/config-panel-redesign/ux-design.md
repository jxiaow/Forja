# 配置面板 UX 设计

## 用户画像

- C++/Qt 开发者，日常使用 VS Code
- 对 VS Code 扩展有基本认知，但不想花时间研究配置
- 期望：装完插件尽快能 build/run

---

## 用户旅程

### 场景 A：首次安装

```
用户装完扩展 → 打开侧边栏 → 看到配置面板

当前体验（差）：
  看到一堆空字段 → 不知道哪些必填 → 试着填 → 不确定填对没 → 焦虑

期望体验（好）：
  看到引导 → 知道只需确认 2 个路径 → 点一下确认 → 完成 → 安心
```

### 场景 B：日常开发

```
用户想切 debug/release 或切项目

当前体验（差）：
  打开面板 → 在"概览"区找 → 没有 mode/arch 切换 → 要去状态栏点
  或者要改 C++ 标准 → 在一堆字段里找

期望体验（好）：
  打开面板 → 第一眼看到项目名和构建参数 → 点一下切换 → 完成
```

### 场景 C：环境出问题

```
用户升级了 Qt 或 VS，构建失败

当前体验（差）：
  看到"概览"里 Qt 状态变红 → 想改路径 → 要展开"环境"折叠区 → 再展开"手动覆盖"

期望体验（好）：
  看到 Qt 状态变红 → 旁边就是路径和修改入口 → 改完立刻变绿
```

### 场景 D：配置远程同步

```
用户第一次配远程同步

当前体验（差）：
  展开远程区 → 看到一大片表单 → 不知道先填什么 → 填完不确定对不对

期望体验（好）：
  展开远程区 → 开关关着，只看到一行说明 → 打开开关 → 渐进展示：先选/加服务器 → 再填路径 → 测试连接确认
```

---

## 设计原则

1. **渐进展示** — 只在需要时展示复杂度；默认状态尽可能简洁
2. **就近操作** — 状态指示和对应操作在同一视觉区域，不跳转
3. **即时反馈** — 每次操作后有明确的视觉响应（颜色变化、提示文字）
4. **最小必要** — 首屏只展示用户最可能需要的信息和操作

## 信息架构

```
面板
├── [条件] 引导卡片
│     目的：首次使用快速完成配置
│     消失条件：环境配置完成
│
├── 项目区 [常驻展开]
│     目的：日常最频繁的操作
│     内容：项目切换、mode/arch、C/C++ 标准、TARGET
│
├── 环境区 [常驻展开]
│     目的：查看环境状态 + 出问题时修改
│     内容：VS/Qt/jom 各一个卡片（状态+路径+覆盖入口）
│
├── 远程同步区 [默认折叠]
│     目的：配置和管理远程同步
│     内容：开关 → 服务器 → 路径 → 测试
│
└── 高级区 [默认折叠]
      目的：低频配置项的收纳
      内容：手动 .pro、RCC、排除目录、提醒开关、IntelliSense
```

## 交互细节

### 引导卡片

```
┌─────────────────────────────────────────────┐
│  ⚡ 快速配置                                 │
│                                             │
│  ● Qt 5.15 (msvc2019)                      │
│    C:\QtCompile\msvc2019-accessible         │
│                                             │
│  ● VS 2022 Community                        │
│    C:\...\Launch-VsDevShell.ps1             │
│                                             │
│  ● jom 可用                                 │
│                                             │
│  [确认使用]              [我要手动配置 →]    │
└─────────────────────────────────────────────┘
```

**状态机：**
- 环境检测中 → 显示 "正在检测环境..." + 加载动画
- 检测完成且有候选 → 显示候选 + 确认按钮
- 检测完成但缺失关键项 → 显示缺失提示 + 引导去环境区手动配
- 用户确认或手动配置完成 → 卡片消失，不再显示

**"确认使用"行为：**
- 将检测到的路径写入 settings
- 卡片淡出消失
- 环境区状态点全部变绿

### 构建参数 — 按钮组

```
┌─────────────────────────────────────┐
│  模式   [ debug ▪ ]  [ release ]    │
│  架构   [ x86 ▪ ]    [ x64 ]       │
└─────────────────────────────────────┘
```

- 当前选中项：实心背景 + 高亮文字
- 未选中项：透明背景 + 暗色文字
- 点击立即切换并保存，无需确认
- 切换时按钮有短暂的过渡动画（150ms）

### 环境卡片 — 合并状态与配置

```
┌─────────────────────────────────────────────┐
│  Qt                              [自动检测] │
│  ● Qt 5.15 (msvc2019)                      │
│  C:\QtCompile\msvc2019-accessible           │
│                                             │
│  [▶ 手动覆盖]                               │
│  ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐  │
│  │ 快速选择  [Qt 5.15 ▾]                │  │
│  │ Qt 路径   [________________] [浏览]   │  │
│  │ Designer  [________________] [浏览]   │  │
│  │ 源码路径  [________________] [浏览]   │  │
│  └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘  │
└─────────────────────────────────────────────┘
```

**状态点颜色：**
- 🟢 绿色：路径有效，工具可用
- 🟡 黄色：路径配置了但未验证 / 检测中
- 🔴 红色：未配置或路径无效

**来源标签：**
- `自动检测` — 灰色标签，表示当前使用的是自动检测到的路径
- `手动配置` — 蓝色标签，表示用户手动覆盖了路径
- `未配置` — 橙色标签，需要用户操作

### 保存反馈

每次配置变更后：
- 输入框边框短暂闪绿（200ms）表示保存成功
- 如果保存失败，边框闪红 + 底部出现错误提示（3 秒后消失）
- 不用 toast/notification，太重；用输入框自身的视觉变化

## 视觉规范

### 间距

- Section 之间：0px（用 header 分隔线区分）
- 卡片之间：12px
- 卡片内字段之间：10px
- 标签与输入框之间：3px

### 字体

- Section 标题：11px, 700, 大写, 0.5px 字间距
- 卡片标题：13px, 600
- 标签：11px, 400, 大写, 暗色
- 输入框：13px
- 提示文字：11px, 暗色
- 状态文字：12px

### 颜色（使用 VS Code 变量）

- 背景：`--vscode-sideBar-background`
- 卡片背景：`rgba(255,255,255,0.03)`
- 输入框：`--vscode-input-background`
- 主按钮：`--vscode-button-background`
- 状态绿：`#22C55E`
- 状态黄：`#F59E0B`
- 状态红：`#EF4444`

### 按钮组样式

```css
.btn-group {
  display: inline-flex;
  border-radius: 4px;
  overflow: hidden;
  border: 1px solid var(--vscode-input-border, rgba(255,255,255,0.1));
}
.btn-group-item {
  padding: 4px 12px;
  font-size: 12px;
  cursor: pointer;
  background: transparent;
  color: var(--vscode-descriptionForeground);
  border: none;
  transition: all 0.15s;
}
.btn-group-item.active {
  background: var(--vscode-button-background);
  color: var(--vscode-button-foreground);
}
```

## 响应式考虑

面板宽度由 VS Code 侧边栏决定（通常 250-400px）：
- 按钮组在窄屏时自动换行
- 字段标签始终在输入框上方（不做左右布局）
- C/C++ 标准两个下拉在宽度够时并排，不够时堆叠

---

## 可访问性

- 所有交互元素可 Tab 聚焦
- 按钮组支持键盘左右箭头切换
- 状态点有对应的 aria-label（"Qt 状态：可用" / "Qt 状态：未配置"）
- 折叠区使用 `<details>` 语义元素
- 颜色不作为唯一信息载体（状态点旁有文字说明）
