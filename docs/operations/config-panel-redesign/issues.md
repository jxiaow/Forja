# 配置面板重新设计

## 现状问题

1. **首次使用无引导** — 打开面板就是一堆空字段，不知道该先填什么、填完了没
2. **信息层级混乱** — "概览"区同时塞了环境状态、项目设置、IntelliSense 生成、高级选项，职责不清
3. **状态与配置分离** — 环境状态 chips 在"概览"，路径配置在"环境"折叠区；用户看到"Qt 未检测"不知道去哪配
4. **远程同步区太重** — 服务器表单、状态、高级设置全挤在一个卡片里，展开后很长
5. **缺少反馈** — 保存成功/失败没有视觉提示；环境检测完成后状态变化不明显

---

## 设计目标

- 首次使用 30 秒内能完成最小配置（确认 Qt 路径 + VS 路径）
- 日常使用只看第一屏（项目名 + 构建参数）
- 低频配置不干扰视线
- 每个状态点旁边就是对应的配置入口，不用跳转

## 关键设计决策

### 1. 引导卡片

- **触发条件**：`qtPath` 为空 且 环境检测已完成 且 检测到了候选
- **行为**：展示检测到的环境，用户点"确认使用"直接写入配置；点"手动配置"跳转到环境区
- **消失条件**：`qtPath` 有值后不再显示

### 2. 构建参数用按钮组替代下拉

- mode 和 arch 只有 2 个选项，用 toggle button group 比 select 更直观
- 当前选中项高亮，点击切换立即保存
- 减少点击次数：下拉需要 2 次点击（打开+选择），按钮只需 1 次

### 3. 环境区：状态和配置合为一体

- 每个工具链是一个卡片，顶部显示状态（绿点 + 版本 + 来源标签）
- 下方显示当前生效路径
- "手动覆盖"折叠在 details 里，大部分用户不需要展开
- 不再有独立的"环境状态 chips"区域

### 4. 远程同步精简

- 开关 + 服务器选择 + 远程路径 是第一层
- 服务器表单和忽略列表折叠在 details 里
- 状态信息（待同步文件数、上次同步时间）紧凑显示在服务器选择下方

### 5. 高级区收纳低频配置

- 手动 .pro 路径、RCC 路径、排除目录、提醒开关
- IntelliSense 生成按钮也放这里（不是日常操作）
- 90% 的用户不需要打开这个区域

---

## 不变的部分

- `messageHandler.ts` 消息协议不变（所有 save* 命令保持兼容）
- `template.ts` 的 `TemplateData` 接口不变（可能新增字段，不删除）
- `settingsStore` / `configService` 数据层不动
- 功能完整性不变，只是重新组织布局
