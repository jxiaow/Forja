# Observability And Ops

## Goal

约束日志、错误、指标、运行态文档和部署/发布操作，确保改动可诊断、可回滚、可交接。

## Core Rules

- 高风险改动要留下可诊断信号。
- 错误日志包含定位上下文，但不泄露密钥。
- 配置变更同步示例和说明。
- 阶段性执行板、验证矩阵、决策记录放运行态文档目录。

## Design Checklist

- 失败时用户或维护者如何定位？
- 是否需要新增日志、指标或错误上下文？
- 是否影响部署、发布或回滚？
- 是否需要运行态文档记录阶段状态？

## Implementation Checklist

- 错误路径有可读上下文。
- 敏感信息没有进入日志。
- 配置示例已同步。
- 运行态文档放在正确目录。

## Common Smells

- 只处理成功路径，没有诊断失败路径。
- 日志输出 token、密码、密钥或完整环境变量。
- 配置改了但示例没改。
- 阶段性 checklist 混进长期开发文档。
