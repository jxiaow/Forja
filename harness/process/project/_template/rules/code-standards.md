## Core Rules

- 遵守仓库已有 formatter 和 lint。
- 复用现有错误模型、日志模型和 helper。
- 不引入未登记依赖。
- 新测试放在仓库既有测试布局中。
- 只在能降低复杂度时新增抽象。

## Design Checklist

- 是否已有同类实现或 helper？
- 是否需要新增依赖？
- 是否改变公共接口或数据格式？
- 测试应该跟随源码、集中放置，还是按功能组织？

## Implementation Checklist

- 格式化和静态检查可通过。
- 新依赖写入正确位置。
- 错误处理符合项目模式。
- 测试命令覆盖改动范围。

## Common Smells

- 为局部逻辑新增重复 helper。
- 混用语言、模块系统或格式风格。
- 捕获错误后丢失上下文。
- 测试放在无法被默认测试命令发现的位置。
