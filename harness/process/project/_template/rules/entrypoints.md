# Entrypoints

## Goal

约束新增能力如何接入运行链路，避免文件存在但没有被应用、构建、测试或发布入口引用。

## Core Rules

- 新能力必须接入对应入口。
- 入口文件只做装配，不承载大段业务逻辑。
- 不删除或重命名公共入口，除非任务明确要求并同步迁移消费者。
- 新依赖必须写入对应包或工作区配置。

## Design Checklist

- 这次改动是否需要新增入口？
- 消费者从哪里发现这个能力？
- 是否影响已有命令、路由、导出或插件注册？
- 最小验证命令是什么？

## Implementation Checklist

- 新文件已被入口引用。
- 入口顺序不会造成冲突或覆盖。
- 公共 API / CLI / 路由 / 插件名保持兼容，或已明确迁移。
- 测试覆盖入口是否可达。

## Common Smells

- 新文件写好了但没有注册。
- 入口文件开始堆业务逻辑。
- 改了导出名但没有更新消费者。
- 只测内部函数，没测入口可达性。
