# Architecture Dependencies

## Goal

约束代码放置位置和依赖方向，避免引入反向依赖。

## Repo Facts

写出目标仓库真实分层、目录职责和依赖方向。

```text
repo-root/
├── layer-a/  # 职责
├── layer-b/  # 职责
└── shared/   # 职责
```

## Core Rules

- 高层可以依赖低层。
- 低层禁止依赖高层。
- 共享层不能反向依赖业务层。
- 入口文件只做组装，不承载大段业务逻辑。

## Design Checklist

- 这段代码的语义归属是什么？
- 是否只是因为当前文件顺手才放这里？
- 是否引入跨层依赖？
- 是否需要接入构建、路由或注册表？

## Implementation Checklist

- import 是否跨越不合理层级。
- 入口文件是否仍只做组装。
- 新文件是否已经接入运行链路。
- 测试是否覆盖被改的边界。

## Common Smells

- 低层直接依赖 UI、路由或应用入口。
- 业务逻辑堆进启动入口。
- 新模块未接入注册入口。
- 为了快速修复破坏既有分层。
