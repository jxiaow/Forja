## Core Rules

- 不删除或重命名已有公共接口，除非任务明确要求。
- 变更接口时同步调用方、文档和测试。
- 数据格式变化要考虑迁移或兼容读取。
- 错误码、退出码和事件名保持稳定。

## Design Checklist

- 这是内部接口还是外部契约？
- 是否需要兼容旧调用方？
- 数据迁移或默认值策略是什么？
- 如何验证调用方仍可工作？

## Implementation Checklist

- 调用方已同步。
- 文档或示例已同步。
- 兼容路径或迁移路径已覆盖。
- 测试覆盖旧输入和新输入。

## Common Smells

- 只改生产者，不改消费者。
- 修改配置格式但不处理旧配置。
- 错误码或事件名随手改名。
- 用临时字段泄露内部实现。
